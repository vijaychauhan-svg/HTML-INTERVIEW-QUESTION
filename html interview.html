Q4. What is DNS and when it comes into picture?

Ans. A.DNS stands for Domain Name System.
B.It works like the phonebook of the internet.
C.Every website is hosted on a server that has an IP address (like 192.168.1.1 or 2001:db8::1).
D.Humans find it hard to remember these numbers, so we use domain names like google.com or youtube.com.
E.DNS translates these human-readable domain names into machine-readable IP addresses.
Example:
When you type www.google.com in your browser:
DNS translates www.google.com ‚Üí 142.250.183.164 (an IP).

2.DNS comes into play whenever you try to access a website or service using a domain name.

Steps (simplified):
A.You type a domain name (e.g., openai.com) in your browser.
B.Your computer asks the DNS resolver (usually your ISP or a configured DNS like Google DNS 8.8.8.8) for the IP address.
C.The DNS resolver checks its cache; if not found, it queries root servers ‚Üí TLD servers (.com, .org, etc.) ‚Üí authoritative DNS servers.
D.It finds the IP address of openai.com.
E.Your browser uses that IP to request the webpage.

Q10.WHAT DOES VIEWPORT META TAG DO?

ANS.The viewport meta tag in HTML is used to control how a web page is displayed on mobile devices and tablets.

Example:

Explanation:

A.width=device-width ‚Üí Sets the page width equal to the device‚Äôs screen width.
B.initial-scale=1.0 ‚Üí Sets the initial zoom level (1 means no zoom).
C.maximum-scale and minimum-scale ‚Üí Control how much users can zoom in or out.
D.user-scalable=no ‚Üí Prevents users from zooming (not recommended for accessibility).

1.Without viewport meta tag:
A.The page might shrink to fit a large "virtual" desktop layout.
B.Text looks tiny, requiring zoom to read.

2.With viewport meta tag:
A.The page adapts to the device screen size (responsive design).
B.Text and elements appear at a readable size automatically.

Q13.what is ip address?

Ans.An IP address (Internet Protocol address) is a unique numerical label assigned to each device (computer, phone, server, etc.) connected to a network (like the internet).
It works like a home address, but for devices on the internet ‚Äî it tells other devices where to find you and how to send information.
Types of IP Addresses:-
1.IPv4 (Internet Protocol version 4)
Format: 192.168.1.1 (4 sets of numbers, each 0‚Äì255).

Example: 142.250.193.78 (used by Google servers). About 4.3 billion unique addresses.
2.IPv6 (Internet Protocol version 6)
Format: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
Created because IPv4 addresses are running out.
Allows trillions of addresses.

Two Main Uses:
>Identifying the device on a network.
>Locating the device so data can be sent to the correct destination.

Example:
A.When you visit www.google.com, your computer:
B.Uses DNS to find Google‚Äôs IP address.
C.Connects to that IP address.
D.Receives the web page data back.

Q.15 HTML5 vs HTML4:-

Ans. A.HTML4: Released in 1997, it was the standard for many years but had limitations for modern web apps.
B.HTML5: Introduced in 2014 (finalized), designed to support modern multimedia, mobile devices, and interactive content.

1. Doctype:-
In HTML4, the doctype declaration was very long and complicated.
In HTML5, it is very short and simple: .

üîπ 2. Multimedia:-
HTML4 could not play audio or video directly; you needed plugins like Flash.
HTML5 has built-in audio and video tags to play media without plugins.

üîπ 3. Graphics and Animations:-
HTML4 relied on Flash or other external tools for graphics.
HTML5 provides canvas and svg for drawing and animations directly in the browser.

üîπ 4. Semantic Elements:-
HTML4 used only generic tags like div and span for structure.
HTML5 introduced meaningful tags such as header, footer, article, section, and nav to make code more understandable.

üîπ 5. Forms:-
HTML4 supported only basic input types such as text, password, and checkbox.
HTML5 added new input types like email, date, number, url, range, and search.

üîπ 6. APIs and Features:-
HTML4 had no built-in APIs.
HTML5 includes many APIs like Geolocation, Local Storage, Offline Web Apps, Drag & Drop, and WebSockets.

üîπ 7. Storage:-
HTML4 used cookies to store data, which had limited capacity and were less secure.
HTML5 introduced localStorage and sessionStorage for larger, safer, and faster data storage.

üîπ 8. Mobile Support:-
HTML4 was not designed for mobile devices.
HTML5 is mobile-friendly, supports responsive design, and works well with modern devices.

Q87.What are logical and physical tags in HTML?

Ans.In HTML, logical and physical tags refer to the distinction between tags that define the meaning/structure of content versus tags that define its appearance/formatting.
1. Logical Tags:-
> Define the meaning or structure of the text rather than its appearance.
> Focus on semantics (what the content represents).
> The browser may render them with a default style, but their main purpose is logical structure.

2. Physical Tags:-
> Define how the text should look visually, regardless of its meaning.
> Focus on presentation, not semantics.
> Mostly deprecated in HTML5, replaced by CSS for styling.
